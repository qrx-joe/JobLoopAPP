# JobLoop AI 项目架构搭建计划 (V1.0)

> 基于 PRD V1.1 文档的技术架构要求制定
> 制定日期：2026-04-22

## 一、技术栈确定

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **前端框架** | Next.js 14+ (App Router) | SSR + API Routes 一体化，适合 MVP 快速迭代 |
| **UI 框架** | React 18 + TypeScript | 类型安全 |
| **样式方案** | Tailwind CSS + shadcn/ui | 组件库，加速开发 |
| **状态管理** | Zustand | 轻量级，适合中等规模 |
| **表单处理** | React Hook Form + Zod | 表单校验 |
| **富文本编辑** | TipTap | 简历在线编辑 |
| **后端/AI API** | Next.js API Routes | 全栈统一，减少运维 |
| **数据库** | Supabase (PostgreSQL) | 免费额度、自带 Auth、实时订阅 |
| **AI 模型** | 多模型路由 | GPT-4o/Claude 3.5 Sonnet/DeepSeek/通义千问 |

## 二、项目初始化计划

### 2.1 Step 1：初始化 Next.js 项目

```bash
# 创建 Next.js 14+ 项目
npx create-next-app@latest jobloop --typescript --tailwind --eslint --app --src-dir --import-alias "@/*" --use-npm

# 进入项目目录
cd jobloop
```

**关键配置选项**：
- TypeScript：✅ 必须
- Tailwind CSS：✅ 必须
- ESLint：✅ 必须
- App Router：✅ 必须（不使用 Pages Router）
- src/ 目录：✅ 必须
- import alias：`@/*`（方便路径引用）

### 2.2 Step 2：安装核心依赖

```bash
# 状态管理
npm install zustand

# 表单处理
npm install react-hook-form @hookform/resolvers zod

# 富文本编辑
npm install @tiptap/react @tiptap/pm @tiptap/starter-kit @tiptap/extension-placeholder

# Supabase 客户端
npm install @supabase/supabase-js @supabase/auth-helpers-nextjs

# AI 调用（OpenAI SDK）
npm install openai

# 文件上传（PDF/Word 解析）
npm install pdf-parse mammoth

# PDF 导出
npm install @react-pdf/renderer

# 工具库
npm install date-fns uuid lodash

# UI 组件（shadcn/ui 后续安装）
npm install class-variance-authority clsx tailwind-merge lucide-react
```

### 2.3 Step 3：初始化 shadcn/ui

```bash
# 初始化 shadcn/ui
npx shadcn-ui@latest init

# 安装基础组件
npx shadcn-ui@latest add button input textarea card dialog tabs toast
```

## 四、数据库 Schema 设计（Supabase）

### 4.1 核心表结构

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email TEXT UNIQUE NOT NULL,
    nickname TEXT,
    avatar_url TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    last_login_at TIMESTAMPTZ,
    -- 免费额度计数
    jd_analysis_count INTEGER DEFAULT 1,
    interview_sessions_count INTEGER DEFAULT 1,
    file_parse_count INTEGER DEFAULT 3,
    pdf_export_count INTEGER DEFAULT 2,
    -- 额度重置日期
    quota_reset_at DATE DEFAULT CURRENT_DATE
);

-- 简历表
CREATE TABLE resumes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    title TEXT NOT NULL,
    type TEXT DEFAULT 'general', -- general, campus, professional
    -- 简历内容 JSON 结构
    content JSONB NOT NULL DEFAULT '{}',
    -- 原始输入记录（便于重新生成）
    original_input TEXT,
    input_mode TEXT DEFAULT 'text', -- text, guided, file, template
    -- 版本控制
    version INTEGER DEFAULT 1,
    is_latest BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- JD 表
CREATE TABLE job_descriptions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    company_name TEXT,
    job_title TEXT NOT NULL,
    source_url TEXT,
    content TEXT NOT NULL,
    parsed_content JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- JD 匹配记录表
CREATE TABLE jd_matches (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    resume_id UUID REFERENCES resumes(id) ON DELETE SET NULL,
    jd_id UUID REFERENCES job_descriptions(id) ON DELETE SET NULL,
    -- 匹配结果 JSON
    result JSONB NOT NULL DEFAULT '{}',
    overall_score INTEGER,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 面试会话表
CREATE TABLE interview_sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    resume_id UUID REFERENCES resumes(id) ON DELETE SET NULL,
    jd_id UUID REFERENCES job_descriptions(id) ON DELETE SET NULL,
    job_title TEXT,
    status TEXT DEFAULT 'active', -- active, completed, abandoned
    current_question_index INTEGER DEFAULT 0,
    -- 实时评分数据
    scores JSONB DEFAULT '[]',
    radar_data JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    ended_at TIMESTAMPTZ
);

-- 面试问答记录表
CREATE TABLE interview_messages (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID REFERENCES interview_sessions(id) ON DELETE CASCADE,
    role TEXT NOT NULL, -- 'user' | 'assistant' | 'system'
    content TEXT NOT NULL,
    metadata JSONB DEFAULT '{}', -- 包含评分、追问类型等
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Prompt 版本表
CREATE TABLE prompt_versions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name TEXT NOT NULL, -- 'resume-generate', 'jd-match', etc.
    version TEXT NOT NULL, -- 'v1.0', 'v1.1', etc.
    prompt_template TEXT NOT NULL,
    variables JSONB DEFAULT '[]',
    is_active BOOLEAN DEFAULT FALSE,
    rollout_percentage INTEGER DEFAULT 0, -- 灰度发布比例
    metrics JSONB DEFAULT '{}', -- 效果指标
    created_by TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 用户反馈表
CREATE TABLE user_feedback (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE SET NULL,
    session_id UUID,
    prompt_version_id UUID REFERENCES prompt_versions(id),
    type TEXT NOT NULL, -- 'thumbs_up', 'thumbs_down', 'report'
    content TEXT,
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Row Level Security (RLS) 策略
ALTER TABLE resumes ENABLE ROW LEVEL SECURITY;
ALTER TABLE job_descriptions ENABLE ROW LEVEL SECURITY;
ALTER TABLE jd_matches ENABLE ROW LEVEL SECURITY;
ALTER TABLE interview_sessions ENABLE ROW LEVEL SECURITY;
ALTER TABLE interview_messages ENABLE ROW LEVEL SECURITY;

-- 用户只能访问自己的数据
CREATE POLICY "Users can only access their own resumes"
    ON resumes FOR ALL USING (auth.uid() = user_id);

CREATE POLICY "Users can only access their own job descriptions"
    ON job_descriptions FOR ALL USING (auth.uid() = user_id);

CREATE POLICY "Users can only access their own matches"
    ON jd_matches FOR ALL USING (auth.uid() = user_id);

CREATE POLICY "Users can only access their own interview sessions"
    ON interview_sessions FOR ALL USING (auth.uid() = user_id);

CREATE POLICY "Users can only access their own messages"
    ON interview_messages FOR ALL
    USING (
        session_id IN (
            SELECT id FROM interview_sessions WHERE user_id = auth.uid()
        )
    );
```

## 五、核心模块实现计划

### 5.1 LLM 调用层（多模型路由）

```
lib/ai/
├── providers.ts          # 模型配置
│   ├── openai: { apiKey, baseURL, defaultModel: 'gpt-4o' }
│   ├── anthropic: { apiKey, defaultModel: 'claude-3-5-sonnet' }
│   └── deepseek: { apiKey, baseURL }
│
├── index.ts             # 统一导出
│   └── callLLM(options): Promise<LLMResponse>
│
└── router.ts            # 路由逻辑
    ├── routeByTask(taskType): string  // 根据任务类型选择模型
    ├── routeByUserTier(tier): string // 根据用户等级选择
    └── fallback(): string            // 降级逻辑
```

**路由策略**：
| 任务类型 | 主模型 | 备选模型 |
|----------|--------|----------|
| 简历生成 | GPT-4o | Claude 3.5 |
| JD 匹配分析 | GPT-4o | DeepSeek |
| 面试问题生成 | Claude 3.5 | GPT-4o |
| 面试追问 | GPT-4o | Claude 3.5 |
| 文件解析（轻量） | DeepSeek | GPT-4o-mini |

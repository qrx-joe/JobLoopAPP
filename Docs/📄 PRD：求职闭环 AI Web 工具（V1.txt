1. 项目概述
1.1 产品名称

求职闭环 AI（可暂定：JobLoop AI / CareerFix / ResumeFlow）

1.2 产品定位

一个基于大模型的求职能力增强Web工具，帮助用户完成从：

零散经历 → 结构化简历 → 岗位匹配 → 面试准备

的完整求职闭环。

【V1.1新增】产品同时具备用户数据持久化能力，支持跨会话的简历管理、JD历史记录与面试训练进度保存。

1.3 核心目标

在最短路径内提升用户求职能力：

将非结构化经历转化为标准简历
提升简历与岗位匹配度
提升面试表达能力
提高面试通过率（核心指标）
【V1.1新增】提高用户重复使用率（通过JD+面试形成使用循环）

1.4 产品本质

不是"简历生成工具"，而是：

👉 求职表达与决策系统（Career Transformation System）

2. 目标用户（用户分层）
2.1 核心用户（Primary ICP）

优先服务人群：

GAP期 6个月–3年
非985/211或普通背景
经历存在但表达混乱
不会写简历 / 不会面试表达
2.2 扩展用户（Secondary ICP）
应届生
转行人群
职业方向不清晰者
海归/回国求职者
2.3 非目标用户（暂不考虑）
顶级名校+强实习背景用户
已有明确offer路径人群
3. 产品核心价值
用户三大核心痛点：
不会总结经历
不会匹配岗位
不会表达面试内容
产品解决：
经历 → 可投递简历
JD → 简历优化方案
简历 → 面试表达能力提升
4. 产品功能结构（MVP版本）
4.1 功能模块一：经历结构化引擎（核心入口）
功能描述

将用户输入的"混乱经历"转换为标准简历内容。

输入：
方式A - 文本自由输入（原有）：
活动经历 / 实习 / 项目 / 学习内容

Step 2：你在其中承担了什么职责？
Step 3：你做了哪些具体的事情？

支持格式：PDF / Word (.docx) / 纯文本

预设模板类型：实习经历模板 / 项目经验模板 / 校园活动模板 / 学习经历模板
每个模板提供填写示例和引导提示
输出：
1️⃣ 简历结构化内容：
项目名称
职责拆解
成果表达（量化）
STAR结构优化
2️⃣ 技能标签：
沟通能力
执行能力

运营能力等
【V1.1新增】技能标签附带置信度评分（高/中/低），低置信度标签提示用户确认
3️⃣ 简历Bullet Points：
可直接复制到简历
UI形态：

左右分栏

左：用户输入
右：实时生成简历
【V1.1新增】引导式模式下为步骤式向导界面（Wizard Step UI）

👉 解决"不会写简历"的问题
👉 【V1.1新增】同时解决"懒得从头写"的问题（文件上传+模板填充）

7.6 【V1.1新增】错误处理与降级机制
LLM调用容错：
超时设置：单次调用最多30秒
重试策略：指数退避，最多重试2次
降级响应：AI不可用时返回基于模板的静态建议内容
输出格式约束：
强制JSON Schema约束LLM输出
前端校验 + 后端二次校验双重保障
校验失败时返回友好错误提示 + 重试选项
用户友好错误提示：
避免展示技术错误码
提供"稍后重试" / "联系客服" / "反馈问题"三个出口
7.7 文件输出：

8. 核心Prompt系统（产品壁垒）
8.1 Prompt工程化架构【V1.1新增】

【V1.1新增】8.1.1 模板化设计原则
所有Prompt采用"基线模板 + 变量注入"模式，禁止硬编码业务逻辑
变量分类：
{user_input} - 用户原始输入
{resume_context} - 已有简历上下文
{jd_context} - 岗位JD内容
{output_format} - 输出格式约束（JSON Schema）
{examples} - Few-shot示例（动态选取）
【V1.1新增】8.1.2 版本管理
每个Prompt模板有独立version_id（如 resume_gen_v1.2）
变更记录：谁在什么时候改了什么
灰度发布：新版本先对10%用户生效，观察效果后全量
回滚机制：效果下降时可一键回退到上一版本
【V1.1新增】8.1.3 缓存策略
相似输入缓存：对用户输入做embedding相似度匹配，复用相近的历史结果（节省API成本 + 提升响应速度）
缓存TTL：简历生成结果24h有效，JD匹配结果JD不变时48h有效
缓存失效：用户手动修改源内容时主动清除相关缓存
【V1.1新增】8.1.4 A/B测试框架
同一功能维护多个Prompt变体
随机分流用户到不同变体
核心评估指标：生成完成率、用户满意度（ thumbs up/down ）、后续转化率
根据数据驱动迭代Prompt质量

15. 【V1.1新增】合规与隐私政策
15.1 数据收集与使用原则
最小化收集：仅收集产品运行所必需的信息
用户知情：所有数据收集项在注册/使用时明确告知用途
用户控制：用户可随时导出/删除自己的全部数据
15.2 个人信息保护（PII处理）
简历中的敏感信息处理规则：
姓名、手机号、邮箱：加密存储，脱敏展示（页面显示"张*"/"138****0000"）
学历、工作经历：正常存储，但不用于任何训练目的
身份证号：禁止采集（如用户误填，系统自动检测并提示删除）
第三方共享：未经用户明确授权，不向任何第三方分享用户简历内容
15.3 AI生成内容声明
【V1.1必做】所有AI生成内容需标注：
简历生成页底部："本内容由AI辅助生成，请核实后使用"
面试回答旁标注："AI建议回答"标识
导出文件水印："Generated with JobLoop AI Assistant"
目的：符合各国AI监管趋势（欧盟AI Act、中国生成式AI管理办法）
15.4 法规合规清单
【V1.1新增】中国法规：
《生成式人工智能服务管理暂行办法》- AI内容需可追溯、可核查
《个人信息保护法》（PIPL）- 用户数据跨境/处理的合规要求
《网络安全法》- 数据安全等级保护
【V1.1新增】国际合规（如面向海外用户）：
GDPR（欧盟）- 用户数据权利（访问权/删除权/携带权）
CCPA（加州）- 消费者隐私权
AI Act（欧盟）- AI系统的透明度义务
15.5 安全技术措施
传输加密：HTTPS/TLS 1.3
存储加密：敏感字段AES-256加密后入库（Supabase支持Row Level Security）
访问控制：基于用户身份的数据隔离（RLS策略）
审计日志：记录所有数据访问操作（谁在什么时间访问了什么数据）
定期安全扫描：依赖漏洞扫描 + 渗透测试（季度）
最后一句关键判断（非常重要）

这个产品的核心不是"AI"，也不是"简历生成"：

👉 而是：让普通人"变得可被录用"

技术可以越来越便宜，但信任越来越贵。
我们要做的不是一个"AI简历工具"，而是一个**值得用户反复回来、甚至愿意推荐给朋友的求职伙伴**。

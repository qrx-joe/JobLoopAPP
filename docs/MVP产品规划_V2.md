# JobLoop V2.0 MVP 产品规划

> 基于职业方向诊断的 GAP 群体求职导航系统
> 核心原则：先帮用户想清楚"能投什么"，再帮用户"怎么投"

---

## 一、产品定位重构

### 原定位（已废弃）

AI 简历生成器 + 面试模拟器

**问题**：假设用户已经知道自己要投什么岗位，只解决"表达优化"问题。

### 新定位（V2.0）

**GAP 期求职者职业方向诊断与经历重组工具**

**核心假设**：目标用户的最大痛点不是"简历写得不好"，而是：

1. 经历杂（转专业、多段学生工作、GAP 期），不知道能投什么岗位
2. 职业方向迷茫（护理→经济→AI→金融？），没有清晰的求职叙事
3. GAP 期长（6 个月–3 年），不知道怎么解释这段空白
4. 没有实习经历，学生工作不知道怎么翻译成职场能力

**产品解决的问题排序**：

```
职业方向诊断（最重要）> 经历重组包装 > 定向简历生成 > JD 体检 > 面试模拟 > 面试复盘
```

## 二、目标用户画像

### 核心用户："迷茫型求职者"

| 维度         | 描述                                                          |
| ------------ | ------------------------------------------------------------- |
| **教育背景** | 非顶尖院校本科（211/普通一本/二本），或顶尖院校但专业不对口   |
| **GAP 状态** | 6 个月–3 年，原因包括：考公/考研/考编失败、疫情待业、转行探索 |
| **经历特征** | 学生工作/社团丰富但缺乏实习，或有多段不相关的短期经历         |
| **技能状态** | 有自学成果（如 AI 工具、编程、证书），但缺乏系统化包装        |
| **心理状态** | 焦虑、自我怀疑、方向混乱，需要情绪价值+结构化引导             |
| **求职城市** | 一线/新一线城市（上海、杭州、深圳、成都、武汉等）             |

### 用户求职旅程中的痛点

```
阶段 1：方向迷茫
"我学经济的，但自学了 AI，还考了证，我能投什么？"

阶段 2：经历混乱
"我的经历太杂了，护理转经济，GAP 两年，HR 看到就 pass 吧？"

阶段 3：简历不会写
"我知道该写什么，但不知道怎么包装成 HR 想看的格式。"

阶段 4：不敢投递
"这个 JD 要求 2 年经验，我能投吗？会不会是 KPI 钓鱼？"

阶段 5：面试失败
"面试完感觉不好，但不知道怎么复盘改进。"
```

**JobLoop V2.0 只解决前 4 个阶段，第 5 阶段放在 P2。**

## 三、MVP 核心功能（P0：3 周内可上线）

### 3.1 经历输入（Experience Intake）

**目的**：零门槛收集用户的全部经历，不预设格式。

**输入方式**：

- 文字输入：自由文本（"我把大学到现在的事都说给你听"）
- 文件上传：PDF/DOCX/TXT（课程论文、活动总结、奖状扫描件等）
- 图片上传：奖状、证书、活动照片（OCR 提取）
- 语音输入：口述经历（Whisper 转文字）

**输出**：

- AI 结构化提取：教育背景、学生工作、项目经历、技能、证书、GAP 期活动
- 不确定性标注："⚠️ 此处信息可能不完整，需要你确认"
- 时间线可视化：让用户看到自己的经历全貌

**技术复用**：复用现有 `lib/file-parser/`（pdf-parse + mammoth）+ 新增图片 OCR。

### 3.2 职业方向诊断（Career Navigator）

**目的**：基于用户经历，推荐适合的城市、行业、岗位类别。

**诊断维度**：

| 维度             | 输出                                                                      |
| ---------------- | ------------------------------------------------------------------------- |
| **能力原型**     | Builder / Analyst / Organizer / Communicator / Tester / Support / Creator |
| **城市推荐**     | 2–3 个城市，附生活成本、竞争程度、目标行业密度                            |
| **岗位类别排序** | 3–5 个岗位方向，附起薪范围、入门门槛、GAP 友好度                          |
| **行业赛道**     | 2–3 个行业，附发展趋势、适合度评分                                        |
| **能力缺口**     | 诚实列出缺少的关键技能/证书，不粉饰                                       |

**关键设计**：

- **必须结合用户背景**：不是通用推荐，是基于 cv.md 的具体分析
- **必须有多选项**："NEVER recommend only one option"
- **必须有 GAP 友好度评分**：每个推荐岗位标注"对 GAP 期容忍度"（高/中/低）
- **必须诚实**：如果用户背景确实不适合某个方向，明确说"不建议"

**技术实现**：

- 复用 `career-navigator.md` 的 7 步逻辑
- Prompt：基于用户经历做原型映射，调用 WebSearch 获取最新市场数据
- 输出：结构化 JSON → 前端渲染为报告页

### 3.3 经历重组包装（Experience Reframer）

**目的**：把杂乱的原始经历，翻译成 HR 能看懂的职场能力。

**包装策略**：

| 原始经历               | 职场翻译                                      |
| ---------------------- | --------------------------------------------- |
| 团支部书记             | 团队管理（30+人）、跨部门协作、活动策划与执行 |
| 学院办公室助理（2 年） | 行政运营、流程优化、多任务并行管理            |
| 转专业（护理→经济）    | 快速学习能力、跨学科适应力、决策执行力        |
| GAP 期自学 AI/完成项目 | 自主驱动力、技术探索能力、全栈开发实践        |
| 校创项目（参与）       | 协作研发、文献调研、成果产出                  |
| 毕业论文优秀           | 研究能力、逻辑分析、学术写作                  |

**GAP 期包装模板**：

- 不推荐："考公考研失败，状态不好"
- 推荐："系统性自学 AI 开发，独立完成 3 个全栈项目，技术栈包括 Vue3/TypeScript/Node.js"

**输出**：

- 经历重组建议（每段经历的新描述）
- GAP 期解释话术（3 个版本：简短版/标准版/详细版）
- 能力标签云（用于简历关键词优化）

### 3.4 定向简历生成（Targeted Resume Generator）

**目的**：同一套经历，针对不同岗位生成不同侧重的简历版本。

**核心功能**：

1. 用户选择目标岗位（从诊断结果中选）
2. AI 生成"一页简历"（中文/英文可选）
3. 支持多版本管理（投 A 公司用版本 1，投 B 公司用版本 2）
4. 简历内容可编辑（AI 生成 → 用户修改 → 确认）

**简历模板设计**：

- 必须一页（A4）
- 模块化：教育背景 / 项目经历 / 技能 / 其他
- 针对 GAP 群体优化：弱化时间线，强化能力证明
- ATS 友好：关键词高亮、标准格式

**技术复用**：

- 复用现有 `/api/resume/generate` 和 `renderPrompt('resume-generate')`
- 扩展 prompt：加入"目标岗位"和"经历重组结果"作为变量
- 前端：复用现有 resume 相关组件

## 四、P1 功能（4–6 周）

### 4.1 JD 体检（Job Description Health Check）

**目的**：在投递前评估岗位的真实性和匹配度，避免无效投递。

**分析维度**（对标 jd.vivi.wiki，但加上个人背景）：

| 维度           | 说明                                               |
| -------------- | -------------------------------------------------- |
| **匹配度评分** | 简历 vs JD 的技能/经验匹配程度                     |
| **GAP 友好度** | 该岗位/公司对 GAP 期的容忍程度（基于 JD 措辞推断） |
| **真诚度**     | JD 是否注水（要求 3 年经验但标"应届生"？）         |
| **风险雷达**   | 996 暗示、薪资模糊、频繁加班词汇                   |
| **黑话翻译**   | "扁平化管理"="没有晋升体系"等                      |
| **建议**       | "值得投"/"谨慎投"/"不建议投"+ 具体原因             |

**差异化**：

- jd.vivi.wiki 是通用分析，JobLoop 结合用户个人背景
- 用户投了某个 JD 后，反馈是否拿到面试 → 训练模型准确率

**技术复用**：

- 复用现有 `/api/jd/match`
- 扩展输出字段：gap_friendly_score, sincerity_score, red_flags[]
- 前端：报告页卡片式展示

## 五、P2 功能（7–10 周）

### 5.1 面试模拟（Mock Interview）

**输入**：简历 + 目标 JD
**输出**：

- 5–10 道面试题（行为/技术/压力/场景）
- 每题附"考察点"和"参考答案框架"
- 支持语音/文字作答
- AI 评分 + 改进建议

### 5.2 面试复盘（Interview Debrief）

**输入**：面试后的回忆（文字/语音/截图）
**输出**：

- 结构化复盘报告
- 逐题评分（1–5 分）
- 表现模式分析（跨多次面试的共性问题）
- STAR+R 故事库更新建议

### 5.3 数据飞轮（Data Flywheel）

**机制**：

- 用户反馈"投了这个 JD 后是否拿到面试/offer"
- 累积数据 → 改进 JD 体检的准确率
- 累积数据 → 改进职业方向诊断的推荐质量

| `src/app/api/jd/match`         | 扩展：增加 gap_friendly_score, sincerity_score 等字段                   |
| `src/app/(main)/resume/new`    | 改造：前置"职业方向诊断"流程，简历生成变为第 4 步                       |
| 新增页面 `/career-navigator`   | 7 步引导式表单，每步数据 localStorage 缓存                              |
| 新增页面 `/experience-reframe` | 经历重组建议页，支持逐条确认/修改                                       |
| 新增页面 `/jd-health-check`    | JD 体检报告页                                                           |

### 7.2 新增依赖

- 图片 OCR：`tesseract.js` 或调用外部 API
- 语音转文字：浏览器 Web Speech API（免费）或 Whisper API（付费）
- 报告导出：`html2canvas` + `jspdf`（前端生成 PDF）

### 7.3 废弃/搁置

- CLI 项目 `career-ops-joeQ`：停止维护，仅作为架构参考
- 微信小程序：P2 以后再考虑
- Supabase 数据库：MVP 阶段继续用 localStorage，数据库在需要用户登录/数据持久化时接入

## 八、差异化竞争点

| 竞品            | 他们做什么           | JobLoop 做什么（不同）              |
| --------------- | -------------------- | ----------------------------------- |
| 超级简历/Canva  | 简历模板+排版        | 不解决"写什么"，只解决"怎么排"      |
| jd.vivi.wiki    | JD 通用分析          | 结合个人背景的 JD 匹配度+GAP 友好度 |
| career-ops      | CLI 工具，给程序员用 | Web 应用，给非技术背景求职者用      |
| 牛客网/leetcode | 刷题+面经            | 解决"投什么"，不是"怎么面"          |
| ChatGPT/Claude  | 通用问答             | 结构化流程+记忆+闭环                |

**JobLoop 的唯一性**：

> **从"经历输入"到"职业诊断"到"经历重组"到"定向简历"到"JD 体检"的完整闭环，专为 GAP 期、非典型背景求职者设计。**

## 九、验证标准（MVP 上线后）

### 9.1 产品验证（内部）

用开发者自己（qrx-joe）作为第一个测试用户：

- [ ] 输入全部大学经历 + GAP 期经历 → AI 提取完整度 > 80%
- [ ] 职业方向诊断 → 推荐岗位包含"产品运营/AI 产品助理/内容运营"
- [ ] 经历重组 → GAP 期被包装为"系统性自学 AI 开发"
- [ ] 定向简历 → 生成 3 个不同岗位版本的简历
- [ ] JD 体检 → 对 10 个真实 JD 的分析准确率 > 70%

### 9.2 市场验证（外部）

- [ ] 10 个目标用户完成全流程，留存率 > 50%
- [ ] 至少 3 个用户凭生成的简历拿到面试
- [ ] 小红书/即刻发布产品体验，自然增长 > 50 个注册用户

## 十、风险与应对

| 风险               | 概率 | 应对                                         |
| ------------------ | ---- | -------------------------------------------- |
| AI 诊断推荐不准确  | 中   | 引入"用户反馈修正"机制，每次推荐后让用户打星 |
| GAP 友好度评估不准 | 高   | 基于用户反馈数据持续训练，初期标注"仅供参考" |
| 用户经历输入太随意 | 中   | 设计引导式问题，降低自由输入门槛             |
| 简历生成质量不稳定 | 中   | 人工审核前 20 份生成结果，调优 prompt        |
| 竞品快速跟进       | 低   | 先发优势+数据飞轮+社区口碑                   |
